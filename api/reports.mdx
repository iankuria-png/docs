---

The reports API gives you programmatic access to all analytics surfaces in Exotic CRM: the sales dashboard, revenue summaries, profile engagement rankings pulled from WordPress, weekly scorecards, team leaderboards, and per-agent activity feeds. All endpoints require a valid bearer token. The dashboard and engagement endpoints are available to the `marketing` role; scorecard, team, and summary endpoints require `admin`, `sub_admin`, or `sales`.

---

## Dashboard summary

Return top-level KPIs, revenue trend, lead funnel, package revenue breakdown, owner performance, and renewal health — all in one response.

```
GET /api/crm/dashboard
```

### Query parameters

<ParamField query="platform_id" type="integer">
  Scope to a specific market. If omitted, data covers all accessible markets.
</ParamField>

<ParamField query="from" type="string">
  Report start date (ISO 8601, e.g., `2026-01-01`). Defaults to 5 months ago.
</ParamField>

<ParamField query="to" type="string">
  Report end date. Defaults to today.
</ParamField>

<ParamField query="reporting_currency" type="string">
  3-letter currency code for normalized totals (e.g., `USD`). Defaults to native currency.
</ParamField>

<CodeGroup>

```bash Request
curl "https://your-crm.example.com/api/crm/dashboard?platform_id=1&from=2026-01-01&to=2026-05-12" \
  -H "Authorization: Bearer {token}"
```

## Profile engagement stats

Return per-profile engagement rankings (views, contact rate, engagement score) pulled from WordPress analytics. Requires selecting a single market.

```
GET /api/crm/reports/profile-engagement
```

### Query parameters

<ParamField query="platform_id" type="integer" required>
  Market to fetch analytics for. Required — engagement data is fetched from WordPress per-market.
</ParamField>

<ParamField query="from" type="string">
  Period start date.
</ParamField>

<ParamField query="to" type="string">
  Period end date.
</ParamField>

<ParamField query="sort_by" type="string" default="engagement_score">
  Sort field: `engagement_score`, `profile_view`, `contact_rate`, or `contact_total`.
</ParamField>

<ParamField query="order" type="string" default="desc">
  `asc` or `desc`.
</ParamField>

<ParamField query="per_page" type="integer" default="25">
  Profiles per page.
</ParamField>

<CodeGroup>

```bash Request
curl "https://your-crm.example.com/api/crm/reports/profile-engagement?platform_id=1&sort_by=profile_view" \
  -H "Authorization: Bearer {token}"
```

```json Response
{
  "filters": { "platform_id": 1 },
  "profiles": [
    {
      "post_id": 1042,
      "name": "Amara",
      "profile_view": 540,
      "contact_total": 62,
      "contact_rate": 11.5,
      "engagement_score": 87,
      "crm_client_id": 501,
      "assigned_agent": { "id": 12, "name": "Jane Ochieng" },
      "subscription_tier": "Premium",
      "subscription_status": "active",
      "wp_profile_url": "https://exotickenya.com/escort/amara/"

}
```

## Export scorecard

Export the weekly scorecard to a file.

```
POST /api/crm/reports/scorecard/export
```

<Note>
Requires `admin`, `sub_admin`, or `sales` role.
</Note>

### Request body

<ParamField body="format" type="string" required>
  Export format: `csv` or `xlsx`.
</ParamField>

<ParamField body="week" type="string">
  ISO week string (e.g., `2026-W19`). Defaults to the current week.
</ParamField>

<CodeGroup>

```bash Request
curl -X POST https://your-crm.example.com/api/crm/reports/scorecard/export \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{"format": "xlsx", "week": "2026-W19"}' \
  -o scorecard_w19.xlsx
```

</CodeGroup>

</ParamField>

<CodeGroup>

## Agent activity feed

Return a paginated chronological feed of actions taken by a specific agent.

```

```

<Note>
Requires `admin` or `sub_admin` role.
</Note>

<ParamField path="user" type="integer" required>

</ParamField>

<ParamField query="entity_type" type="string">
  Filter by entity: `client`, `lead`, `deal`, or `payment`.
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number.
</ParamField>

<CodeGroup>

```bash Request
curl "https://your-crm.example.com/api/crm/team/12/activity?entity_type=deal" \
  -H "Authorization: Bearer {token}"
```
