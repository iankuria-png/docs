# Exotic Online University

This section exists for one reason: **no one can effectively sell, support, market, or improve a product they don't fully understand.**

The pages in this tab explain how the platform works from the **escort's point of view** — from the moment they first hear about Exotic to the moment they renew their subscription. Each stage is documented with the actual system behaviour, department-specific notes, FAQs, troubleshooting steps, and certification questions so that every team member can speak confidently about the product.

<Info>
  **Who this is for:** All internal staff — Sales & Customer Service, Finance, SEO, and IT. The content is internal only and should not be shared directly with customers.
</Info>

---

## Who is the customer?

The Exotic customer (referred to throughout as the **escort**) is an independent adult service provider who lists their profile on one of Exotic's WordPress-powered advertising markets (ExoticKenya.com and similar).

| Attribute | Details |
|---|---|
| **Primary goal** | Get their profile visible to potential clients, generate contact enquiries, and earn income |
| **Primary concern** | Is my profile live? Am I getting contacts? Is my payment confirmed? |
| **Tech literacy** | Moderate — comfortable with smartphones and mobile money (M-Pesa), less familiar with web dashboards |
| **Payment preference** | M-Pesa (STK Push or Till number), occasionally card via hosted checkout |
| **Contact channels** | WhatsApp (primary), phone call, SMS, Viber; Telegram coming soon |
| **Relationship with Exotic** | Vendor–platform — Exotic hosts and promotes their profile in exchange for a subscription fee |

## End-to-end journey map

The escort lifecycle has nine stages. The pilot documentation (fully written, with SOPs, FAQs, and certification) covers **Stage 5 — Subscription, Payment & Activation** because it is the most cross-departmental and revenue-critical. All other stages are documented as structured stubs with "interview pending" callouts for sections that require staff input to complete.

| Stage | What happens | Documentation status |
|---|---|---|
| **1 — Discover** | Escort learns about Exotic via referral from another escort, an outbound sales call, or organic Google search | [Stub →](/product/stage-1-discover) |
| **2 — Register** | Escort creates a WordPress account and basic profile on the public site | [Stub →](/product/stage-2-register) |
| **3 — Profile setup & media** | Escort fills in bio, rates, city, contact details, and uploads photos/videos | [Stub →](/product/stage-3-media) |
| **4 — Verification** | Staff verify the escort's identity and age via WhatsApp ID photo; CRM verified toggle is set | [Stub →](/product/stage-4-verification) |
| **5a — Subscription selection** | Escort chooses a package and duration on the self-checkout page | [Full →](/product/stage-5a-subscription-selection) |
| **5b — Payment initiation** | Escort pays via M-Pesa STK Push or hosted checkout | [Full →](/product/stage-5b-payment-initiation) |
| **5c — Payment confirmation** | Payment provider callback confirms receipt; payment is matched to client | [Full →](/product/stage-5c-payment-confirmation) |
| **5d — Profile activation** | CRM provisions the subscription and WordPress publishes the profile | [Full →](/product/stage-5d-profile-activation) |
| **5e — Failures & recovery** | What goes wrong, how to detect it, and how to fix it | [Full →](/product/stage-5e-failure-modes) |
| **6 — Customer contact** | A site visitor contacts the escort via WhatsApp, phone, SMS, or Viber | [Stub →](/product/stage-6-contact) |
| **7 — Renewal** | Subscription nears expiry; CRM sends reminders; escort repurchases | [Stub →](/product/stage-7-renewal) |
| **8 — Upgrades** | Escort moves to a higher package tier | [Stub →](/product/stage-8-upgrades) |

## Department touchpoint matrix

Use this matrix to understand where your team's work intersects with each customer stage.

| Stage | Sales & CS | Finance | SEO & IT |
|---|---|---|---|
| Discover | Outbound calls, referral tracking | — | SEO ranking drives organic arrivals |
| Register | Guide escort through sign-up; catch failed registrations | — | WordPress account creation health |
| Profile setup | Review completeness score in CRM; prompt escorts to upload photos | — | Photo quality + profile fields affect search ranking |
| Verification | **Own this stage** — collect ID, verify age, set CRM toggle | — | — |
| Subscription selection | Answer package questions; handle objections | Aware of package pricing and currency | Payment link providers configured per market |
| Payment initiation | Send payment link if escort needs help; retry STK Push failures | Monitor initiated payments in recovery queue | STK Push gateway health; provider routing |
| Payment confirmation | Answer "did my payment go through?" questions | **Own this stage** — reconciliation, reversals, settlement | Callback endpoint health; webhook signature failures |
| Profile activation | Confirm profile is live; handle "my profile isn't showing" tickets | — | WP sync endpoint health; activation failures |
| Customer contact | — | — | SEO: profile visibility; IT: contact channel integrations |
| Renewal | **Own this stage** — run campaigns, make calls, close renewals | Track renewal revenue per market | — |
| Upgrades | Handle upgrade requests; apply discounts with approval | Approve upgrade pricing changes | Tier change provisioning |

## Subscription packages

The following tiers are available on most markets. Exact pricing is configured per market in Settings → Integrations.

| Tier | Typical benefits | Target escort |
|---|---|---|
| **Basic** | Standard listing, limited photos | New escorts testing the platform |
| **Featured** | Highlighted in search results, more photos | Established escorts wanting more exposure |
| **Premium** | Top placement, full photo gallery, priority support | High-volume escorts |
| **VIP** | Premium placement + additional visibility boosts | — |
| **VVIP** | Maximum visibility, all features | Top-tier escorts |
